# CodeAtlas 学习增强功能设计规格

## 概述

在现有 MVP 基础上新增三个面向初学者的学习增强功能：选中代码一键解释、函数悬浮提示、执行流追踪。

## 核心理念

自由探索 + 即时解惑：用户随意浏览代码，遇到不懂的地方随时获得 AI 解释，无需跳转或切换上下文。

## 功能设计

### 1. 选中代码一键解释

#### 触发方式
- Monaco 编辑器中**选中任意代码**（函数名、一行、多行）
- 选区旁出现**浮动按钮**「AI 解释」或按快捷键 `Cmd+E`

#### 数据流
```
Monaco 选中文本 → 拼接上下文 → DeepSeek API (stream) → AIPanel 流式渲染
```

#### UI 行为
- 选中后右侧 AI 面板自动切换为"代码解释"视图
- 显示：选中代码片段（高亮）+ AI 流式解释
- 完成后显示"返回文件概览"按钮
- 内容不覆盖之前的对话，独立展示
- 选中文本 > 300 字符时截断，附带前后 5 行上下文

#### 降级
- 未配置 API Key 时：显示 Mock 静态解释（预生成的关键函数解释）

### 2. 悬浮提示 (Hover Tooltip)

#### 触发方式
- Monaco 中鼠标悬停在**函数名**上 400ms

#### 内容
- 函数名 + 签名
- 一句话作用描述
- 所属文件 + 行号
- "查看详情 →" 链接（跳转 AI 面板）

#### 数据来源
- 签名 → `symbols.json` 的 `signature` 字段
- 作用描述 → 新增 `symbols.json` 每条 symbol 的 `description` 字段
- 文件/行号 → `symbols.json` 已有

#### 实现
- Monaco `HoverProvider` API
- 每次切换文件时重新注册 provider
- 匹配当前 token 是否在已知函数列表中

### 3. 执行流追踪

#### 触发方式
- 代码视图或图谱中，右键函数 →「追踪执行流」

#### UI：底部追踪面板

```
┌──────────────────────────────────────────┐
│  📍 面包屑: func1 → func2 → func3  [关闭] │
├──────────────────────────────────────────┤
│  Step 2/3: func2                         │
│  ┌──────────────────────────────────────┐│
│  │ 伪代码 (AI 生成)                     ││
│  │   ↓ 检查参数                          ││
│  │   ↓ 调用 func3()  ← 高亮下一步        ││
│  │   ↓ 返回结果                          ││
│  └──────────────────────────────────────┘│
│                                           │
│  [◀ 上一步]  [自动跟随: 开关]  [下一步 ▶]  │
└──────────────────────────────────────────┘
```

#### 功能规格

| 元素 | 行为 |
|------|------|
| 面包屑 | 显示已追踪的调用路径，点击任意节点跳回 |
| 代码卡片 | 当前函数的 AI 伪代码摘要，高亮下一跳 |
| 自动跟随 | 开启后，代码区点击任意调用自动推进 |
| 上/下一步 | 沿调用链前进/后退 |
| Monaco 联动 | 推进时自动跳转到对应函数源码位置 |
| 关闭/重置 | 点击关闭或新追踪起点自动重置 |

#### 数据依赖
- 调用链 → `calls.json`
- 源码 → `public/kernel-source/`
- AI 伪代码 → DeepSeek API，结果缓存到 localStorage

### 4. 数据补充

`symbols.json` 每条记录新增 `description` 字段：

```json
{
  "name": "schedule",
  "kind": "function",
  "file": "kernel/sched.c",
  "line": 34,
  "signature": "void schedule(void)",
  "description": "内核调度器主函数，选择下一个可运行的进程并切换上下文"
}
```

## 技术实现要点

| 组件 | 文件 | 说明 |
|------|------|------|
| 选中解释按钮 | CodeViewer 内联 | Monaco `addAction` 或选区浮动按钮 |
| Hover Provider | CodeViewer 内联 | `languages.registerHoverProvider` |
| 追踪面板 | 新建 `app/components/trace/TracePanel.tsx` | 底部可折叠面板 |
| 流式渲染 | `ai-api.ts` | 新增 `chatCompletionStream` 函数 |
| 数据生成 | `scripts/generate-descriptions.ts` | 批量生成 symbol descriptions |

## 不做的

- 预设学习路线（用户选择自由探索）
- 阅读进度/书签（用户未选）
- 笔记系统（用户未选）
- 概念标注（用户未选）
